E-commerce Backend en Go con SAGA + Outbox

Demo mínima para mostrar un flujo event-driven real: crear orden, reservar stock y resolver estado final.

GitHub Backend health

Cómo funciona (resumen)

  1. POST /orders crea orden en estado PENDING.
  2. Order guarda evento en outbox.
  3. Relay publica order.created en RabbitMQ.
  4. Inventory responde inventory.reserved o inventory.failed.
  5. Order actualiza estado final: CREATED o FAILED.
